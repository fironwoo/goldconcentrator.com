# GoldConcentrator.com 轻量动效与 CMS 内容管理设计

## 目标

在不引入第三方动画库的前提下，为全站增加克制、专业的滚动入场效果；同时完善 Sveltia CMS，使网站所有者可以逐步上传封面图片、正文图片、技术文章、项目案例和视频，并确保已发布内容能在前台被访问。

## 范围

本次工作包括：

- 全站通用滚动入场动效。
- 首页重点区域的分组、依次入场效果。
- 方案、项目、文章和视频的封面图片字段。
- 公开的视频列表页和视频详情页。
- 视频草稿过滤。
- 中文后台操作指南。

本次不包括：

- 视差滚动、粒子效果、3D 动画或自动播放背景视频。
- 自建视频存储和转码。
- 自动把普通 YouTube 链接转换为 Short 或其他平台链接。
- 在 CMS 中直接修改页面布局和颜色。

## 动效设计

### 技术方案

使用原生 CSS 和一个全局 `IntersectionObserver`。动效逻辑由独立客户端脚本负责，并通过 `BaseLayout` 在所有前台页面加载。

页面元素通过稳定的选择规则自动加入动效，不要求内容编辑者在 CMS 中填写动画设置。主要目标包括：

- 页面主标题、摘要和行动按钮。
- `.section-heading`。
- `.surface-card`、方案卡片和项目卡片。
- 流程步骤、文章列表项和 FAQ。
- 正文中的二级、三级标题、图片和引用块。

### 视觉参数

- 默认初始位移：向下 `16px`。
- 默认透明度：`0` 到 `1`。
- 动画时长：约 `520ms`。
- 缓动：平稳减速，不使用弹跳。
- 同组项目延迟：每项约 `60ms`，设置合理上限，避免长列表等待。
- 动画每次页面加载只执行一次。

首页首屏采用轻微的加载入场效果，不等待滚动观察器，以避免首屏出现空白。卡片原有悬停效果保留。

### 可访问性和降级

- 当系统启用 `prefers-reduced-motion: reduce` 时，取消位移、延迟和动画。
- JavaScript 未加载或浏览器不支持 `IntersectionObserver` 时，所有内容立即可见。
- 动效不得影响键盘焦点、链接点击、表单校验和页面布局。
- 动效只使用 `opacity` 和 `transform`，避免触发布局重排。

## CMS 内容模型

### 图片

Sveltia CMS 继续将上传文件保存到：

- 仓库目录：`public/images/uploads`
- 公开路径：`/images/uploads/...`

以下集合增加可选的封面图字段：

- 矿种方案。
- 项目案例。
- 技术文章。
- 视频。

封面字段包含图片路径和替代文字。正文 Markdown 编辑器仍可插入多张图片。

推荐规则：

- 文件名使用小写英文、数字和连字符。
- 封面图优先使用横向照片。
- 不把图库素材描述为真实客户项目。
- 图片替代文字说明画面内容，不堆砌关键词。

### 视频

视频继续使用 YouTube 托管。CMS 保存标题、摘要、矿种、YouTube 地址、发布日期、封面、是否重点展示、草稿状态和补充正文。

新增：

- `/videos/`：仅列出 `draft: false` 的视频。
- `/videos/<slug>/`：视频详情页，显示可嵌入播放器、说明和相关项目入口。
- 主导航或知识区域提供视频入口。

YouTube 地址必须经过安全解析，只接受受支持的 YouTube 视频地址；无法识别时不生成播放器，并显示外部观看链接。

### 发布状态

CMS 保持 `editorial_workflow`：

- 草稿：内容仍在编辑，不出现在正式构建。
- 审核中：等待确认。
- 已发布：合并到 `main` 后触发 Cloudflare Pages 自动部署。

视频的 `draft` 字段是额外的前台保护。即使文件已经发布到仓库，只要 `draft: true`，前台视频列表和详情构建都不会公开该视频。

## 后台操作指南

新增中文指南文档，覆盖：

1. 打开 `/admin/` 并使用 GitHub 登录。
2. 新建或修改方案、案例、文章和视频。
3. 上传封面和在正文中插入图片。
4. 保存草稿、提交审核和发布。
5. 修改、撤回和删除内容。
6. 图片尺寸、命名、替代文字和压缩建议。
7. YouTube 链接和视频草稿使用方法。
8. 标题、摘要、日期、分类和 SEO 的填写建议。
9. 常见问题，包括发布后等待部署、图片不显示和登录失败。

指南同时保存在仓库中，并在 `/admin/` 登录页提供简短入口说明，方便后续查阅。

## 测试与验收

自动测试覆盖：

- 动效脚本识别目标、分组延迟和降级逻辑。
- YouTube 地址解析。
- 草稿视频过滤。
- 内容模型接受可选封面及替代文字。

构建验证覆盖：

- Astro 类型检查和生产构建成功。
- Sitemap 包含公开视频页面，不包含草稿视频。
- 所有现有页面继续生成。

浏览器验证覆盖：

- 首页滚动时标题、卡片和流程依次入场。
- 内页正文、图片和卡片正确入场。
- 手机宽度下无横向滚动或内容遮挡。
- 减少动态效果时内容立即可见。
- `/admin/` 能加载中文 CMS。
- 视频列表和详情页可访问，草稿不公开。

## 发布方式

代码和文档提交到 GitHub `main` 后，由现有 Cloudflare Pages 项目自动构建和部署。正式域名 DNS 仍保持不变，先在 `goldconcentrator-com.pages.dev` 完成验收。
