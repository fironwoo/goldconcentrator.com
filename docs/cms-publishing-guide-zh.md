# GoldConcentrator.com 内容后台操作指南

本指南用于日常上传图片、技术文章、项目案例、矿种方案和 YouTube 视频。后台地址为正式网址或预览网址后面加 `/admin/`。

## 一、登录后台

1. 打开 `/admin/`。
2. 点击“使用 GitHub 登录”。
3. 使用已经获得 `fironwoo/goldconcentrator.com` 仓库权限的 GitHub 账户登录。
4. 第一次登录时，GitHub 可能要求确认授权。确认仓库名称正确后再授权。
5. 登录后，左侧会显示“全局设置、矿种方案、项目案例、技术文章、视频”。

不要把 GitHub 密码、验证码或恢复代码发给其他人。后台只需要 GitHub 授权，不会要求提供 Cloudflare、Zoho 或 Resend 密码。

## 二、发布流程

后台使用审核工作流：

1. **保存草稿**：内容仍在编辑，访客看不到。
2. **设为审核中**：内容已经基本完成，等待检查事实、图片授权和英文表达。
3. **发布**：内容写入 GitHub `main` 分支。
4. **等待部署**：Cloudflare Pages 自动重新生成网站，通常需要几分钟。
5. **检查网页**：打开对应页面，确认标题、图片、段落和链接正常。

点击发布后网页不会立即变化。请先等待 Cloudflare 部署完成，再刷新页面。不要因为暂时看不到变化而连续重复发布。

## 三、上传图片

### 封面图片

方案、案例、文章和视频都可以填写“封面图片”和“封面替代文字”。上传封面后必须填写替代文字，否则网站构建会拒绝发布。

推荐规格：

- 尺寸：`1600 × 900` 像素。
- 比例：`16:9` 横图。
- 格式：优先 WebP，其次 JPEG。
- 单张图片尽量控制在 `500 KB` 以下。
- 文件名使用小写英文、数字和连字符，例如 `alluvial-gold-trommel-test.webp`。
- 不要使用空格、中文括号或微信自动生成的长文件名。

合适的替代文字示例：

`Alluvial gold trommel during wet commissioning in Jiangxi`

不要写成堆砌关键词的广告文字，例如：

`best gold mining machine cheap price`

### 正文图片

1. 在“正文”Markdown 编辑器中，将光标放在需要插图的位置。
2. 点击图片按钮。
3. 上传图片或选择已经上传的图片。
4. 填写图片说明。
5. 保存草稿并使用预览检查位置。

所有后台图片会保存到 `public/images/uploads/`。删除文章不一定会自动删除它使用过的图片，所以删除内容前应确认图片是否仍被其他页面使用。

### 图片真实性

- 客户项目图片必须取得允许发布的授权。
- 供应商工厂图片没有书面授权时，不得写成 `our factory`。
- 图库、AI 生成图或氛围图不得描述为真实客户矿山、真实试验或公司工厂。
- 案例中的产能、品位、回收率和结果必须有证据及适用条件。

## 四、发布技术文章

1. 进入“技术文章”并点击“新建文章”。
2. 填写英文标题。
3. 填写摘要与 SEO 描述，建议约 120 至 155 个英文字符。
4. 填写分类，例如 `Alluvial gold`、`Hard rock gold` 或 `Gravity concentration`。
5. 选择发布日期。
6. 如为重点文章，打开“重点文章”。
7. 上传封面并填写替代文字。
8. 在正文中使用清晰的二级标题组织内容。
9. 保存草稿，检查后再提交审核和发布。

文章应回答实际问题。不要为了 SEO 重复生成国家、吨位或设备页面，也不要在正文堆砌关键词。

## 五、发布项目案例

1. 进入“项目案例”并新建案例。
2. 填写国家；如需保密，可写 `Country withheld`。
3. 填写矿种、处理量和流程概述。
4. 在“证据说明”中写明数据来源、授权状态或仍需核验的内容。
5. 只有证据和图片授权都完成后，才打开“证据已核验并允许收录”。
6. 上传获授权的封面图并填写替代文字。
7. 正文建议包括：原始问题、已知矿况、试验依据、流程选择、交付范围和结果条件。
8. 保存草稿，提交审核后再发布。

`verified` 没有打开时，案例页会带有不允许搜索引擎收录的标记。未核验案例不得宣传确定的回收率或客户结果。

## 六、修改矿种方案

矿种方案字段较多：

- `SEO 标题` 用于搜索结果标题。
- `排序` 决定方案顺序。
- `页面类型` 不要随意更改。
- `关键变量` 说明哪些矿况会改变流程。
- `典型流程阶段` 应描述工艺逻辑。
- `设备作用` 应解释设备职责，不写成商城清单。
- `常见问题` 回答客户真正关心的问题。

现有正式网址已经固定。不要随意修改文件名或 slug，否则旧链接可能失效。

## 七、发布视频

视频文件不直接上传到网站。先把视频上传到 YouTube，再在后台保存视频资料。

支持的地址：

- `https://www.youtube.com/watch?v=...`
- `https://youtu.be/...`
- `https://www.youtube.com/shorts/...`

操作步骤：

1. 进入“视频”并新建。
2. 填写标题、摘要和关联矿种。
3. 粘贴完整 YouTube 地址。
4. 选择发布日期。
5. 上传获授权的封面图；如不上传，网站会尝试使用 YouTube 缩略图。
6. 在补充说明中写明矿石、设备、试验或现场背景。
7. 初次编辑时保持“视频草稿”为开启状态。
8. 完成事实和授权检查后，关闭“视频草稿”。
9. 再执行审核和发布。

只要 `draft: true`，即使文件已经进入 GitHub，视频也不会出现在 `/videos/`，也不会生成公开详情页。

## 八、修改、撤回和删除

### 修改已发布内容

打开内容并点击编辑，修改后重新提交审核和发布。等待 Cloudflare 部署完成后检查网页。

### 暂时撤回视频

把“视频草稿”重新打开并发布修改。下一次部署后，视频列表和详情页会停止公开。

### 删除内容

1. 确认该页面不是重要外部链接的目标。
2. 确认正文图片是否仍被其他页面使用。
3. 在后台选择删除。
4. 审核删除变更并发布。
5. 部署后检查旧网址是否返回 404。

删除已被 Google 收录的正式页面前，应先判断是否需要重定向到相关页面。

## 九、发布前检查表

- 标题和摘要是英文且没有明显拼写错误。
- 图片有使用授权。
- 封面图片和替代文字成对填写。
- 案例数字有来源和适用条件。
- 没有虚构客户、奖项、工厂所有权或回收率。
- YouTube 地址属于支持格式。
- 视频不准备公开时仍保持草稿。
- 邮箱、WhatsApp 和地址没有误写。
- 页面预览在手机上可读。

## 十、常见问题

### GitHub 登录失败

确认登录账户已获得仓库权限。退出其他 GitHub 账户后重新登录。不要重新创建同名仓库。

### 图片上传后不显示

检查是否已经发布，而不只是保存草稿；检查封面替代文字是否填写；等待 Cloudflare 部署完成后强制刷新浏览器。

### 发布后网站没有变化

打开 Cloudflare Pages 查看最新部署状态。部署仍在进行时继续等待；部署失败时不要反复发布，先查看失败日志。

### YouTube 视频不能播放

确认地址属于支持格式，视频不是私密状态，并且 YouTube 允许嵌入。无法识别的地址不会生成播放器。

### 构建提示封面字段错误

封面图与封面替代文字必须同时填写或同时留空。

### 不确定内容是否可以公开

先保存草稿，不要发布。优先确认客户授权、工厂授权和数据证据。
